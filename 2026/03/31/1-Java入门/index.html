3 Java语言平台概述

3.1 Java平台介绍

  • javaSE
    • java语言的标准版,是java语言的核心。可以做一些桌面应用程序。
  • javaME
    • java语言的小型版,是为开发一些电子设备产品和嵌入式设备提供的方案。
  • javaEE
    • java语言的企业版，为企业开发环境下的应用程序提供解决的方案。该体系中包含技术比如:servlet.主要针对的是web应用程序开发

3.2 JRE和JDK

JRE(Java Runtime Environment):java运行环境，其中包含了java虚拟机(JVM)和java程序所需的核心类库。如果想要运行一个开发好的java程序，计算机只需要安装JRE即可。

JDK(Java Development Kit):java开发工具包，是JDK提供给java开发人员使用的，其中包含了java的开发工具，也包含了JRE。

JRE:JVM + 核心类库

JDK:JRE + 开发工具

JVM:java虚拟机，能够保证java语言的跨平台特性。

3.3 Java语言的跨平台特性

  • 平台指操作系统:windows、linux、mac
  • 跨平台:java程序可以在任意的操作系统上运行。一次编写，到处运行。
  • 原理:不同的平台提供了不同的JVM

java语言是跨平台的，JVM不是跨平台的

5.2 编写HelloWorld代码

编写HelloWorld代码需要遵守java语言的语法规范

文件名称必须是HelloWorld.java,修改完后缀后，使用jdk包中的bin文件夹下的javac.exe来打开HelloWorld.java

这个过程称之为编译,编译是会检查java语法，如果符合java语法，会生成一个.class后缀的文件，称之为字节码文件

然后再使用jdk包中的java.exe执行HelloWorld.class文件

这个过程称之为叫运行,会把java代码转换成计算机认识的二进制代码，执行结果并输出到屏幕上

5.3 入门案例的问题

  • 中英文符号的问题—一定要使用英文符号
  • 严格按照大小写的规范去编写
  • 类名和文件名要匹配

5.4 文件命名规则补充

  • 公共类的名称要和文件名一致(公共类就是class前面添加public的类)
  • 一个java文件中只能有一个公共类,但是可以有多个普通类(普通类就是不用public修饰的类)
  • java文件中无论是公共类还是普通类都会通过javac编译生成一个对应的.class文件
  • 在java语言中是严格区分大小写的，但是windows系统中不区分大小写，所以如果在java中在不区分大小写的情况下类名相同，那么在windows中只会产生一个对应的.class文件
  • 在java中类名是每个单词的首字母都大写 HelloWorld TestDemo

8 关键字

关键字是被java语言赋予特殊含义，具有专门用途的单词。比如之前接触的public class static void都是java已经预设好的关键字。

关键字的特点:

  • java中的关键字全部都是小写字母组成
  • 关键字在IDE中都会有特殊的高亮效果

9 字面量

在程序执行的过程中，其值不能发生改变的量。

  • 字符串常量 用双引号括起来的内容
    • “HelloWorld” 可以没有字符，也可以有多个字符
  • 整数常量 所有的整数
  • 小数常量 所有的小数
  • 字符常量 用单引号括起来的内容
    • ‘A’ ‘中’
    • 只能有一个字符
  • 布尔常量 true 和 false
  • 空常量 null(数组那天再讲)

11 常见码表

  • ASCII码表总共包含128个，包含了英文大小写，阿拉伯数字，常用符号
  • ISO8859-1总共 有256个，前128位还是ASCII码表
  • GB2312(GBK)–使用两个字节表示一个汉字
  • 建立一个Unicode编码体系–中国通过这套标准有一套UTF-8码表 使用三个字节表示一个汉字
  • 所有的码表都包含ASCII码表，无论是GBK还是UTF-8,一个英文字母还是占一个字节
